# Dashboard 产品需求文档（PRD）定稿

## 一、已确定要点

| 项目 | 内容 |
|------|------|
| 产品形态 | 演示/原型型 Dashboard 网站 |
| 数据 | Mock 数据 |
| 实现路径 | Figma 解析 → React/HTML 设计规格 → Flutter Web 复刻，设计美学一致 |
| 设计来源 | [Nucleus UI - LITE](https://www.figma.com/design/OVIHrj6SJXrDQ3IT2SzJlc/…?node-id=858-8369)（– Dark 画布） |
| 核心模块 | 侧边栏导航、顶栏、KPI 卡片、图表、数据表格、表单/筛选区 |

## 二、PRD 1～8 项（合理默认）

### 1. 页面与路由

- **单页 Dashboard**：侧栏 + 主内容区，通过侧栏切换“视图”即可，不启用多页面路由。
- 主内容区视图：**概览**（默认）、**数据报表**、**设置**（占位）。
- 视图切换为前端状态切换，无 URL 路由。

### 2. 侧边栏

| 菜单项 | 说明 |
|--------|------|
| 概览 | 默认页，展示 KPI + 图表 + 表格入口摘要 |
| 数据报表 | 图表 + 表格 + 筛选区 |
| 设置 | 占位页，仅标题与一句说明 |

图标：用 Material/通用图标（dashboard、bar_chart、settings）或与 Nucleus 风格接近的线性图标。

### 3. 顶栏

- **必有**：面包屑（如：概览 / 当前子页）、当前用户（头像 + 名称）、通知图标。
- **不加**：全局搜索、主题切换、多语言（保持 Demo 简洁）。

### 4. KPI 卡片

共 4 个，置于概览/数据报表顶部：

| 标题 | 含义 | 单位/格式 |
|------|------|-----------|
| 总用户数 | 累计注册用户 | 整数，如 12,345 |
| 今日订单 | 当天订单数 | 整数，如 128 |
| 收入 | 可选为“今日收入”或“本月收入” | 货币，如 ¥8,888 |
| 完成率 | 任务/订单完成率 | 百分比，如 94.5% |

### 5. 图表

- **折线图** 1 个：近 7 日访问量（横轴日期，纵轴次数）。
- **柱状图** 1 个：近 5 日订单量（横轴日期，纵轴订单数）。
- **饼图** 1 个：某分类占比（如：订单状态：已完成 / 进行中 / 已取消，Mock 比例）。

### 6. 表格

- **业务含义**：订单列表（或“最近订单”）演示。
- **列**：订单号（文字）、金额（货币）、状态（标签：已完成/进行中/已取消）、日期（yyyy-MM-dd）。
- 行数：Mock 约 5～10 条，支持表格上方筛选后“刷新”列表（前端过滤即可）。

### 7. 表单/筛选区

- **位置**：顶栏下、表格上方；仅在「数据报表」视图中展示。
- **筛选项**：时间范围（开始日期、结束日期）、状态（下拉：全部/已完成/进行中/已取消）、关键词（订单号或备注，可选）。
- **按钮**：查询、重置。

### 8. 技术约束

- **目标平台**：Flutter **Web** 优先；结构预留移动端扩展，本期不实现 iOS/Android。
- **依赖**：无强制要求；使用 Flutter 官方 Material + 通用图表库（如 fl_chart 或 similar），无 getx/bloc/dio 等除非你后续补充。

## 三、设计规范（来自 Figma / Nucleus – Dark）

- **主色**：`rgb(107, 78, 255)` → `#6B4EFF`
- **深色背景**：`rgb(9, 10, 10)` → `#090A0A`
- **白色/高对比文字**：`rgb(255, 255, 255)`
- **次要文字**：`rgb(151, 156, 158)`
- **浅灰/分割**：`rgb(218, 218, 218)`
- **字体**：Inter（或系统 sans-serif 降级）
- **圆角**：主按钮 48px（胶囊）；卡片/输入框等可用 8～12px。
- **字号**：大标题 32px/36px 行高；正文 16px/24px 行高；辅助 12px/16px 行高。
